# Product Vision Document
**Role:** Product Owner
**Project:** Audiris — Corporate Website
**Version:** 1.0
**Date:** 2026-03-30

---

## 1. Executive Summary

Audiris is an IT and software consultancy company with roots in Quality Assurance and Automation & Performance Testing, now offering a broader portfolio including Software Development and Outsourcing. This document defines the vision, goals, audience, and success criteria for the Audiris corporate website.

---

## 2. Problem Statement

Audiris currently lacks a digital presence that:
- Communicates the company's evolved service portfolio to prospects
- Differentiates Audiris from generic software agencies by leading with its QA-first heritage
- Generates qualified inbound leads from target client segments
- Builds credibility and trust with enterprise buyers and growth-stage companies

Without this, Audiris relies entirely on referrals and outbound sales, limiting scalable growth.

---

## 3. Product Vision Statement

> **For** technology leaders, engineering managers, and procurement teams at startups, mid-size companies, large enterprises, and software agencies —
> **Who need** reliable software quality assurance, test automation, software development, or outsourcing expertise —
> **The Audiris website** is a professional digital presence and lead-generation platform —
> **That** clearly communicates Audiris's services, credentials, and unique QA-rooted identity, converting visitors into qualified enquiries —
> **Unlike** generic IT consultancy websites that lead with technology stacks or headcount, **Audiris leads with quality** as a founding philosophy, not an afterthought.

## 5. Target Audiences & Personas

### Persona 1 — Startup CTO / VP Engineering
| Attribute | Detail |
|-----------|--------|
| **Name** | Alex, 34, CTO at a 30-person SaaS startup |
| **Pain** | Shipping fast but accumulating technical debt and bugs; needs QA without a full-time hire |
| **Goal** | Find a reliable testing partner who understands modern CI/CD |
| **Decision trigger** | A production incident or failed release |
| **What wins them** | Speed, automation-first thinking, flexible engagement, transparent pricing |
| **Desired action** | Book a discovery call |

### Persona 2 — Enterprise Procurement / IT Director
| Attribute | Detail |
|-----------|--------|
| **Name** | Maria, 47, IT Director at a financial services firm |
| **Pain** | Compliance requirements, manual testing backlog, legacy system risk |

| **Decision trigger** | Failed audit, regulatory deadline, internal headcount freeze |
| **What wins them** | Credentials, case studies, ISO/certifications, enterprise references |
| **Desired action** | Download capability deck / contact sales |

| Attribute | Detail |

| Attribute | Detail |
|-----------|--------|

## 6. Business Goals & Success Metrics

| Goal | KPI | Target (6 months post-launch) |
|------|-----|-------------------------------|
| Generate inbound leads | Contact form submissions | ≥ 10/month |
| Book discovery calls | Calendly bookings | ≥ 5/month |
| Establish organic presence | Google Search impressions | ≥ 1,000/month |
| Reduce bounce rate | Bounce rate | < 55% |
| Build brand credibility | Avg. session duration | > 2 min |
| Reach target audience | Organic keyword rankings | Top 20 for 5 target terms |

### Target SEO Keywords (initial)
- "QA consultancy" / "QA consulting company"
- "Test automation services"
- "Software testing outsourcing"
- "Performance testing services"
- "Software development outsourcing"

## 8. Scope

### In Scope (v1 — Launch)
- Home page
- 4 Service pages (QA & Testing, Automation Testing, Software Development, Outsourcing)
- About page
- Contact page
- Basic SEO (meta tags, sitemap, structured data)
- Contact form with email delivery
- Cookie consent banner
- Google Analytics / Plausible integration
- Mobile responsive design

### Out of Scope (v2+)
- Blog / Resources section
- Case studies (placeholder page only)
- Client portal
- Live chat
- Multi-language support
- CMS integration

## 9. Assumptions & Constraints

| Type | Description |
|------|-------------|
| **Assumption** | No client logos or testimonials available at launch — placeholder strategy required |
| **Assumption** | No CMS needed at launch — content managed via code |
| **Assumption** | One developer (using Claude Code) building the site |
| **Constraint** | Budget: minimal — use free/free-tier services where possible |
| **Constraint** | Timeline: MVP launch within 4–6 weeks |
| **Constraint** | No backend server — static site with serverless form handling |
