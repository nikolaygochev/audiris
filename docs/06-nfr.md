# Non-Functional Requirements (NFR)
**Role:** Business Analyst
**Project:** Audiris — Corporate Website
**Version:** 1.0
**Date:** 2026-03-30

---

## 1. Performance

| ID | Requirement | Target | Measurement |
|----|-------------|--------|-------------|
| NFR-P01 | Lighthouse Performance score | ≥ 90 (mobile & desktop) | Lighthouse CI |
| NFR-P02 | Largest Contentful Paint (LCP) | < 2.5s | Core Web Vitals |
| NFR-P03 | Cumulative Layout Shift (CLS) | < 0.1 | Core Web Vitals |
| NFR-P04 | First Input Delay / INP | < 200ms | Core Web Vitals |
| NFR-P05 | Total page weight (home) | < 500 KB (compressed) | Browser DevTools |
| NFR-P06 | All images | WebP format, `next/image` with lazy loading | Code review |
| NFR-P07 | Fonts | Loaded via `next/font` (no layout shift) | Visual inspection |
| NFR-P08 | JS bundle | Tree-shaken; no unused dependencies shipped | Bundle analyser |

## 2. Accessibility

| ID | Requirement | Standard | Tool |
|----|-------------|----------|------|
| NFR-A01 | WCAG 2.1 Level AA compliance | All pages | axe DevTools / Lighthouse |
| NFR-A02 | Colour contrast ratio (text on background) | ≥ 4.5:1 (normal text), ≥ 3:1 (large text) | Contrast checker |
| NFR-A03 | All interactive elements keyboard navigable | Tab, Enter, Space, Escape | Manual testing |
| NFR-A04 | All images have descriptive `alt` attributes | Decorative images: `alt=""` | Code review |
| NFR-A05 | Focus indicators visible | Ring visible on all focusable elements | Visual inspection |
| NFR-A06 | Form fields have associated `<label>` elements | WCAG 1.3.1 | Code review |
| NFR-A07 | Motion reduced when `prefers-reduced-motion` is set | All Framer Motion animations disabled | Browser setting test |
| NFR-A08 | Semantic HTML structure | Correct heading hierarchy (h1→h2→h3) per page | axe |
| NFR-A09 | ARIA labels on icon-only buttons | E.g. hamburger menu | Code review |

## 3. SEO

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-SEO01 | Unique `<title>` per page | Format: "[Page] | Audiris" (max 60 chars) |
| NFR-SEO02 | Unique meta description per page | 120–160 characters, includes target keyword |
| NFR-SEO03 | One `<h1>` per page | Includes primary keyword for that page |
| NFR-SEO04 | Open Graph tags | og:title, og:description, og:image, og:url on all pages |
| NFR-SEO05 | Structured data (JSON-LD) | Organization schema on home; Service schema on service pages |
| NFR-SEO06 | Canonical URLs | `<link rel="canonical">` on all pages |
| NFR-SEO07 | sitemap.xml | Auto-generated by Next.js; all public routes included |
| NFR-SEO08 | robots.txt | Allow all; reference sitemap |
| NFR-SEO09 | No broken links | All internal links return 200; verified pre-launch |
| NFR-SEO10 | Image alt text | All non-decorative images have keyword-relevant alt text |
| NFR-SEO11 | URL structure | Clean, lowercase, hyphenated: `/services/qa-testing` |
| NFR-SEO12 | Mobile-first indexing | Site fully functional and readable on mobile |

## 4. Security

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-SEC01 | HTTPS enforced | Vercel enforces HTTPS by default; HTTP → HTTPS redirect |
| NFR-SEC02 | Security headers | `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, `Permissions-Policy` via `next.config.js` |
| NFR-SEC03 | Content Security Policy (CSP) | Restrictive CSP header configured; inline scripts avoided |
| NFR-SEC04 | API route protection | `/api/contact` validates all inputs; rejects malformed requests |
| NFR-SEC05 | No secrets in client-side code | Resend API key stored in `.env.local`; never exposed to browser |
| NFR-SEC06 | Spam protection | Honeypot field on contact form; silently discard bot submissions |
| NFR-SEC07 | Dependency audit | `npm audit` run before launch; no high/critical vulnerabilities |
| NFR-SEC08 | No sensitive data logged | Form submissions must not be logged in server logs |

## 5. Compatibility

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-C01 | Desktop browsers | Chrome 120+, Firefox 120+, Safari 17+, Edge 120+ |
| NFR-C02 | Mobile browsers | Chrome for Android, Safari for iOS (last 2 major versions) |
| NFR-C03 | Minimum mobile width | 375px (iPhone SE) — no horizontal scroll |
| NFR-C04 | Responsive breakpoints | sm: 640px, md: 768px, lg: 1024px, xl: 1280px (Tailwind defaults) |
| NFR-C05 | Touch targets | All interactive elements ≥ 44×44px on mobile |
| NFR-C06 | JavaScript disabled | Core content readable (Next.js SSG ensures this) |

## 6. Reliability & Availability

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-R01 | Uptime | ≥ 99.9% (Vercel SLA) |
| NFR-R02 | Contact form fallback | If Resend API fails, show fallback email address to user |
| NFR-R03 | Static generation | All pages pre-rendered at build time (Next.js SSG) — no runtime server dependency |
| NFR-R04 | Deployment | Zero-downtime deploys via Vercel (atomic deployments) |

## 7. Maintainability

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-M01 | Content in constants | All copy stored in `/lib/content.ts` — no hardcoded strings in JSX |
| NFR-M02 | Component reuse | Shared layout components for service pages; no copy-paste pages |
| NFR-M03 | TypeScript strict mode | `strict: true` in `tsconfig.json` |
| NFR-M04 | No inline styles | All styles via Tailwind classes; no `style={{}}` except dynamic values |
| NFR-M05 | README | Project README with: setup instructions, environment variables, deploy steps |
| NFR-M06 | Environment variables documented | `.env.example` with all required keys and descriptions |

## 8. Analytics & Observability

| ID | Requirement | Detail |
|----|-------------|--------|
| NFR-AN01 | Page views tracked | All page views tracked after consent |
| NFR-AN02 | Contact form conversion tracked | Form submission event fired to analytics on success |
| NFR-AN03 | Outbound link tracking | LinkedIn and email clicks tracked |
| NFR-AN04 | No PII in analytics | Names, emails must not appear in analytics event data |
